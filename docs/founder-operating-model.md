# Founder Operating Model

The founder is the first operating system of the company.

In Agent-Fabric, the founder role combines builder, product manager, product strategist, product owner, platform steward, ecosystem designer, and governance authority until the organization has enough structure to distribute those responsibilities safely.

## Definition

The Founder Operating Model defines how the founder shapes the platform, protects coherence, drives product strategy, validates customer outcomes, curates community contribution, and governs the transition from founder-led creation to organization-led execution.

```text
Founder
  -> builder
  -> product manager
  -> product strategist
  -> product owner
  -> platform steward
  -> ecosystem designer
  -> governance authority
```

## Core rule

```text
Founder creates coherence before the organization scales execution.
```

The founder is not only writing code or making strategy.
The founder holds the product truth until the platform, community, partners, and governance model can sustain it.

---

## Founder as Builder

The founder builds the first working reality.

Responsibilities:

```text
prototype core ideas
create first implementation paths
connect architecture to code
remove ambiguity
prove the loop works
protect simplicity
make the system runnable
```

Builder rule:

```text
Do not outsource the first principles.
```

---

## Founder as Product Manager

The founder translates user pain into product decisions.

Responsibilities:

```text
define users
define outcomes
write product requirements
prioritize work
shape workflows
convert feedback into issues
protect usability
track adoption signals
```

Product manager rule:

```text
Every feature must map to a user outcome.
```

---

## Founder as Product Strategist

The founder defines the market thesis and product direction.

Responsibilities:

```text
position the platform
choose wedge markets
define service delivery model
define partnership model
align pricing with outcomes
identify category language
protect differentiation
sequence the roadmap
```

Product strategy rule:

```text
Do not sell components. Sell governed outcomes.
```

---

## Founder as Product Owner

The founder owns the backlog and product acceptance boundary.

Responsibilities:

```text
accept or reject features
define done
protect kernel-native architecture
approve roadmap changes
rank issues
merge directionally important work
reject incoherent expansion
```

Product owner rule:

```text
Not every contribution becomes product.
```

---

## Founder as Platform Steward

The founder protects the platform identity and coherence.

Responsibilities:

```text
protect vocabulary
protect protocol boundaries
protect kernel invariants
protect organizational autonomy
protect platform-owned contracts
protect community contribution quality
protect partner/provider scope
```

Stewardship rule:

```text
Open contribution does not mean ownerless direction.
```

---

## Founder as Ecosystem Designer

The founder designs the partnership network.

Responsibilities:

```text
define vendor/provider/partner roles
define capability graph
define certification model
define commitment model
define evidence obligations
align partners to service outcomes
```

Ecosystem rule:

```text
Vendors supply. Providers commit. Partners co-deliver. Platform owns the contract.
```

---

## Founder as Governance Authority

The founder establishes the first governance boundary.

Responsibilities:

```text
define authority model
define risk appetite
define decision rights
define release gates
define community rules
define partner admission rules
define escalation rules
```

Governance rule:

```text
The organization is the autonomy boundary.
```

---

## Transition model

Founder-led does not mean founder-bottleneck forever.

The transition should be:

```text
Founder-led creation
  -> maintainer-assisted contribution
  -> community-supported implementation
  -> partner-supported delivery
  -> organization-led governance
```

The founder keeps direction while delegating execution.

## Decision filter

Every major decision should pass this filter:

```text
Does it preserve kernel-native truth?
Does it improve the service delivery model?
Does it support platform-owned contracts?
Does it make community contribution easier?
Does it help partners/providers fulfill scoped commitments?
Does it make the product more usable?
Does it protect AGenNext identity?
```

## One-line definition

The founder is the first builder, product manager, product strategist, product owner, platform steward, ecosystem designer, and governance authority who turns Agent-Fabric from vision into a coherent, community-contributed, platform-owned service delivery system.
